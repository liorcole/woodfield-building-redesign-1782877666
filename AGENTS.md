# Woodfield Building Redesign Visual Guide

## North Star

Redesign Woodfield Building Services as a highly legitimate, quietly premium construction portfolio. It should feel like an architectural design studio spent months on the composition, motion, image sequencing, and interaction details.

The experience should feel technical, established, clean, precise, and calm. It should not feel like a generic builder template, a stock contractor brochure, or a loud local-trade website.

## Reference Translation

Use the Taste Labs reference direction as a principle source, not as something to copy:

- Dark charcoal hero sections with warm off-white editorial sections.
- Small, precise navigation and labels.
- Staged native imagery as a product/object system.
- Thin-line diagrams, measured rows, and modular archive views.
- Horizontal media rails, drag interactions, and scroll choreography.
- Quiet table-like credibility and service information.

## Woodfield Adaptation

Translate the style into construction and refurbishment:

- Use Woodfield's real project photography as the visual proof.
- Treat service categories as a technical capability system.
- Treat project galleries as an architectural archive.
- Use the 1988 founding date, Cannock location, insurance-backed guarantees, local authority approval, FMB/TrustMark credibility, testimonials, and phone/email details as trust anchors.
- Keep CTAs practical: call, email, request a conversation, view projects.

## Visual System

- Dominant colors: charcoal, near black, warm off-white, soft white, muted grey.
- Accent: restrained construction cyan/blue taken from the old Woodfield identity, used sparingly.
- Typography: precise sans-serif, no novelty trade fonts.
- Corners: 8px or less for panels and buttons.
- Imagery: real native photos, cropped intentionally, never blurred into generic atmosphere.
- Motion: staged, smooth, functional, and reduced-motion safe.

## Interaction System

Include premium interaction design:

- Scroll reveals and image mask reveals.
- Pointer-reactive hero object stack on desktop.
- Draggable project/media rails.
- Lightbox/modal for project imagery.
- Magnetic CTA buttons.
- Mobile menu with clean focus states.
- No motion that blocks contact, booking, or reading.

## Content Rules

- Preserve native content and claims from the live site.
- Make every public old route available locally.
- Do not link back to old Woodfield pages.
- Use local downloaded native images.
- Use `tel:` and `mailto:` links for phone and email.
- Use a static contact form that opens the visitor's email client with the supplied details.

## Quality Bar

Before finishing:

- Desktop and mobile must both feel designed.
- No text overlap, horizontal overflow, or broken internal routes.
- No internal link should point to `woodfieldbuilding.co.uk`.
- Every native public page should have a redesigned local route.
- The first viewport should immediately communicate established construction craft, not a generic hero.
